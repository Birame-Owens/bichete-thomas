RAPPORT PERFORMANCE PRODUCTION - NDEYA SHOP
Date: 12 mai 2026

ETAT ACTUEL
Le site NDEYA SHOP est stable apres les derniers correctifs.
Le test de charge principal est valide avec 100 utilisateurs virtuels pendant 6 minutes.

Resultat de reference k6:
- 100 VUs pendant 6 minutes
- 12 742 requetes traitees
- 0% erreurs HTTP
- 100% des checks reussis
- p95: environ 826 ms
- max: 1.72 s
- environ 35 requetes/seconde

Conclusion:
La performance serveur est bonne pour une charge moderee a forte.
Le site peut etre considere comme pret pour la production actuelle.

- Aucun echec HTTP pendant le dernier test k6.
- Toutes les pages testees ont repondu en moins de 2 secondes.
- Les pics de lenteur observes avant ont disparu.

CE QU'IL FAUT CONTINUER A SURVEILLER
1. Temps de reponse

2. Logs Laravel

3. Serveur
Surveiller:
- CPU

- Redis healthy


AMELIORATIONS PRIORITAIRES
1. Index base de donnees
Ajouter des index cibles pour garder les performances quand les donnees vont augmenter:
- produits visibles par categorie
- produits populaires
- nouveautes
- sitemap
- recherche texte avec pg_trgm si PostgreSQL

2. Recherche
Les recherches avec ILIKE "%mot%" peuvent ralentir quand il y aura beaucoup de produits.
Solution recommandee:
- activer pg_trgm PostgreSQL
- ajouter index GIN trigram sur produits.nom, produits.description, categories.nom

3. Nombre de vues produit
Eviter d'ecrire en base a chaque visite produit si le trafic augmente.
Solution recommandee:
- stocker temporairement les vues dans Redis/cache
- synchroniser en base toutes les quelques minutes

4. Monitoring
Ajouter une surveillance externe:
- UptimeRobot, Better Stack ou equivalent
- alerte si le site tombe
- alerte si temps de reponse trop long
- alerte si erreur 500/502/504

5. Lighthouse mobile
Faire un test PageSpeed/Lighthouse mobile chaque mois.
Verifier surtout:
- LCP
- CLS
- poids des images
- temps de chargement checkout

6. Sauvegardes
Verifier que les backups fonctionnent vraiment:
- backup base PostgreSQL
- backup storage/app/public
- test de restauration au moins une fois


CHECKLIST APRES CHAQUE DEPLOIEMENT
1. Verifier que le site s'ouvre:
https://ndeya-shop.site

2. Verifier les containers:
docker ps

3. Verifier les logs:
docker logs --tail=100 backend
docker logs --tail=100 nginx
docker logs --tail=100 queue

4. Lancer les caches Laravel:
php artisan optimize:clear
php artisan config:cache
php artisan route:cache
php artisan view:cache

5. Si des images ont ete modifiees:
php artisan images:optimize

6. Relancer un test de charge apres gros changement:
k6 run ndeya-load-test.js
